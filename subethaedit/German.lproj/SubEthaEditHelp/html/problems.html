Fehlerbeseitigung
 

Wenn Sie Hilfe bei einem Problem benötigen, das bei der Verwendung von SubEthaEdit auftritt, finden Sie nachstehend Lösungen.

In bin der einzige Macintosh-Benutzer im Subnetz. Wo kann ich die kollaborative Textbearbeitung ausprobieren?
Verbinden Sie sich mit dem offiziellen CodingMonkeys Test Server indem Sie hier klicken.

Ist SubEthaEdit 2.x kompatibel mit der vorangegangenen Version?
Nein. Aufgrund seines wesentlich weiter entwickelten Netzwerk-Protokolls kann sich SubEthaEdit 2.x nicht mit vorangegangenen Programmversionen verständigen. Jedoch ist das neue Protokoll erweiterbar und bietet somit Raum für zukünftige Entwicklungen.

Ich sehe keine freigegebenen Dokumente? Was läuft falsch?
Falls Sie die in Mac OS X eingebaute Firewall aktiviert haben müssen Sie Ports 6942 bis 6951 öffnen, um SubEthaEdit die Netzwerk-Kommunikation zu erlauben. Gehen Sie dazu wie folgt vor: Öffnen Sie "Systemeinstellungen → Sharing → Firewall". Klicken Sie "Neu...", um eine Einstellung hinzuzufügen. Wählen Sie als Dienst "Andere" und geben Sie "6942-6951" als Bereich und "SubEthaEdit" als Beschreibung an.

Ich befinde mich hinter einem NAT-Router. Wie kann ich mmit anderen über das Internet zusammenarbeiten, die sich ebenfalls hinter einem NAT-Router befinden?
Ein Benutzer muß an seinem NAT-Router Port-Weiterleitung aktivieren. Die Ports 6942 bis 6951 müssen vom NAT-Router zum Computer weitergeleitet werden. Nun sollte der andere Benutzer eine Verbindung aufbauen können, die nun beiden Seiten zum Freigeben von Dokumenten genutzt werden kann. Wenn Sie mit Ihrem Web-Browser http://whatismyip.com laden, können Sie Ihre dynamisch zugewiesene IP-Adresse herausfinden.
 Bitte die Seite "Informationen über NAT" anzeigen.
