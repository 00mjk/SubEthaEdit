1. BEEP session creation

1.1 Using Rendezvous

A Rendezvous browser object browses the local subnet for the given service "_hydra._tcp.". When a found service is resolved its TXT record is parsed to retrieve the user identifier hosting the service. If the user identifier is not currently kown a TCP connection in the context of a BEEP session is made to this service.
Both peers greet each other with following profiles:

http://www.codingmonkeys.de/BEEP/SubEthaEditHandshake
http://www.codingmonkeys.de/BEEP/TCMMMStatus
http://www.codingmonkeys.de/BEEP/SubEthaEditSession

After the greeting has been exchanged the initiator of the session starts a channel with the "Handshake" profile.

C: MSG 0 1 . 52 135
C: Content-Type: application/beep+xml
C:
C: <start number='1'>
C:   <profile uri='http://www.codingmonkeys.de/BEEP/SubEthaEditHandshake' />
C: </start>
C: END
S: RPY 0 1 . 221 105
S: Content-Type:application/beep+xml
S:
S: <profile uri='http://www.codingmonkeys.de/BEEP/SubEthaEditHandshake' />
S: END

When the channel is open the client sends a handshake message to the server and the server replies itself with a handshake.

C: MSG 1 0 . 0 77
C: GRTuserid=B5E8CFF7-68A8-11D8-BAC5-000A95A55D58\001version=2.00\001rendez=vousEND
S: RPY 1 0 . 0 77
S: GRTuserid=95809D1A-6CF2-11D8-9341-000A95A55D58\001version=2.00\001rendez=vousEND

Now the client acknowledges the handshake with a ACK message which ...

C: MSG 1 1 . 77 3
C: ACKEND
S: ???
S: ???

A status channel is now opened.

C: MSG 0 1 . 52 135
C: Content-Type: application/beep+xml
C:
C: <start number='2'>
C:   <profile uri='http://www.codingmonkeys.de/BEEP/TCMMMStatus' />
C: </start>
C: END
S: RPY 0 1 . 221 105
S: Content-Type:application/beep+xml
S:
S: <profile uri='http://www.codingmonkeys.de/BEEP/TCMMMStatus' />
S: END
