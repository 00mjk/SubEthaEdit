解決問題
 

如果您想要幾決在使用 SubEthaEdit 時遇到的問題，請參考下列的解決方法。

內部網路裡面只有我一個人用麥金塔，何處能讓我測試協同編輯？
按這裡連到 CodingMonkeys 的官方測試伺服器。

SubEthaEdit 2.0 和 1.x 相容嗎？
不相容。由於牽涉許多關於網路協定的問題，SubEthaEdit 2.0 不能跟 1.x 的版本連線。然而，新的通訊協定擁有極佳的擴充性，這對我們未來的版本發展有助益。

我看不到任何一份共享出來的文件，哪裡不對了？
請檢查您是否把防火牆打開了。如果是，請允許 SubEthaEdit 使用埠號 6942 到 6951 的範圍。如果您是使用 Mac OS X 內建的防火牆，請參照下列的步驟來更改設定：打開 “系統偏好設定 → 共享 → 防火牆”. 按下允許埠號清單旁邊的 “新增”，接著在傳輸埠名稱選擇 “其他”，並在傳輸埠號碼的欄位輸入 “6942-6951”，然後在描述的地方輸入 "SubEthaEdit" 即可。

我的麥金塔是位在路由器之後。我要如何與其他 Internet 的使用者一起參與編輯，並且繼續讓麥金塔待在路由後方？
有人必須要打開路由的埠號轉送（port forwarding）功能。埠號 6942 到 6951 必須要從路由轉送到網路內部的麥金塔上。設定完畢後，其他使用者就可以與您的麥金塔建立連線，即讓雙方都可以參與編輯共享的文件。如果您想要知道您的動態 IP 位置，您可以連到 http://whatismyip.com 查詢。
